Two-Stage Geometric Audit for Malicious Client Detection

Geometric audit view for thesis use: the verifier first screens clients by directional consistency using the FIP score threshold εdir, then performs a robust magnitude audit on the surviving provisional set using the median, MAD, adaptive contamination rate, and an acceptance interval [τmin(t), τmax(t)]. The output is the accepted encrypted set 𝒜(t) and rejected tag set 𝒟(t).

Figure. Editable academic-style geometric diagram for Algorithm 4. The verifier starts with an empty provisional pass set P(t) and rejected set 𝒟(t). In the first stage, it scans the FIP score map 𝒮(t) and rejects every client whose score satisfies si(t) ≤ εdir, recording the reason as Direction; all others are placed into the provisional pass set P(t). If no client survives this stage, the algorithm immediately returns ∅ and 𝒟(t). Otherwise it forms X(t) from the surviving scores, computes the robust center μ(t) = Median(X(t)) and the robust spread MAD(t) = max(Median(|x − μ(t)|), ε). It then estimates the contamination rate ρ(t), constructs the adaptive lower threshold τmin(t) = α(ρ(t))μ(t) and upper threshold τmax(t) = μ(t) + γmadMAD(t), and finally accepts only those encrypted submissions 𝒰(t)[TagIDi] whose scores lie in the interval [τmin(t), τmax(t)]. Clients outside this band are added to 𝒟(t) with reason Magnitude. The output is the accepted encrypted set 𝒜(t) together with the full rejected-tag set 𝒟(t). Here εdir is chosen near zero so that negatively aligned or weakly aligned updates are removed before magnitude screening.
