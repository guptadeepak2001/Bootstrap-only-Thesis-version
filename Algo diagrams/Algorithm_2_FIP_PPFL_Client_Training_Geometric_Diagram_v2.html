Client-Side Local Training and Encrypted Gradient Submission

Geometric client-side view for thesis use: each client initializes its local state from the round model, performs local mini-batch training, averages the accumulated gradient, flattens it into coordinates, encodes and encrypts each coordinate, and submits the encrypted message Mi(t) to the FIP Verifier.

Figure. Editable academic-style geometric diagram for Algorithm 2. Client Ci receives the round model θ(t), the Paillier public key pk, fixed-point scale S, and local hyperparameters. It initializes the local model θi, zeroes the running gradient accumulator gi(t) and mini-batch counter mi, and performs E epochs of local mini-batch training over the client dataset Di. For each mini-batch 𝓑 ⊂ Di with |𝓑| = B, the local gradient h = ∇θiL(fθi; 𝓑) is computed, accumulated into gi(t), counted via mi, and used to update the local model by θi ← θi − ηh. After local training, the client averages the accumulated gradient, flattens it into coordinates (gi,1(t), …, gi,d(t)), encodes each coordinate with the fixed-point map EncodeS(·) to obtain ĝi,k(t), encrypts it under Paillier as [[ĝi,k(t)]], packages the anonymous tag together with the encrypted list into the message Mi(t), and sends only this encrypted submission to the server-side FIP Verifier.
