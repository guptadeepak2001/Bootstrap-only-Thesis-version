Homomorphic Functional Inner Product Score Computation

Geometric server-side view for thesis use: the FIP Verifier receives encrypted client submissions and the trusted reference, encodes the reference coordinates, computes each homomorphic functional inner-product score in encrypted form, sends only the encrypted scalar accumulator to the TKA for decryption, decodes the scalar score, and returns the score map 𝒮(t).

Figure. Editable academic-style geometric diagram for Algorithm 3. The FIP Verifier begins by initializing the round score map 𝒮(t) = ∅ and encoding the trusted reference coordinates as R̂k(t) ← EncodeS(Rk(t)) for k = 1, …, d. For each encrypted submission (TagIDi, {[[ĝi,k(t)]]]}k=1d) in 𝒰(t), it initializes the encrypted scalar accumulator [[ŝi(t)]] ← Encpk(0) and updates it coordinate-wise by the Paillier homomorphic rule [[ŝi(t)]] ← [[ŝi(t)]] · ([[ĝi,k(t)]])R̂k(t) mod n2. Only this encrypted scalar accumulator is sent to the TKA, where ŝi(t) ← Decsk([[ŝi(t)]]) and the real-valued score is recovered by si(t) ← DecodeS²(ŝi(t)). The verifier then stores the result as 𝒮(t)[TagIDi] ← si(t) and returns the completed FIP score map. For negative encoded reference coefficients, Paillier exponentiation is implemented using ciphertext inversion, i.e. c^a = (c^{-1})^{|a|} mod n^2 when a < 0.
