5.3.4     Phase IV: Secure Aggregation

Phase IV is short. The ciphertexts that the audit rejected are simply dropped;
they are never decrypted at all. Whatever survives is folded into one pooled
encrypted sum, and only that sum is later opened.

5.3.5     Phase V: Model Update and Distribution

In Phase V the server decrypts only the aggregate update, averages it across the
accepted clients, and applies the global model step. There is one corner case
worth calling out: if the audit rejects every client in a round, the model is simply
held unchanged for the next round. Algorithm 5 combines the secure aggregation
step and the model update in one block, while the bootstrap reference remains
fixed.
