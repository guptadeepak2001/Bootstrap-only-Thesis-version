FIP-PPFL System Initialization and Reference Bootstrapping

Clean geometric architecture view for thesis use: key generation and client registration, model/verifier initialization, and public-data reference bootstrapping into a unit trusted direction gref(0).

Figure. Editable academic-style geometric diagram for Algorithm 1. The TKA generates the Paillier key pair and keeps sk sealed while releasing only pk and the fixed-point scale S. The server initializes θ(0) ← InitModel(fθ) (zero initialization for the logistic-regression experiments) together with U(0) = ∅, broadcasts θ(0) to the registered clients, and the server-side FIP Verifier stores the initial trusted reference direction. The trusted public bootstrap set Dinit produces the raw gradient g̃ref(0), which is normalized to gref(0) using max(||g̃ref(0)||2, ε). The bottom inset shows the geometric normalization from a raw gradient vector to a unit direction in a 2-D slice of the parameter space.
