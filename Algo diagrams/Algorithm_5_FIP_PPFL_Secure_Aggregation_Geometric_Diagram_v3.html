Secure Aggregation and Momentum-Based Reference Update

Geometric aggregation view for thesis use: if the accepted encrypted set is empty, the server keeps the current model and reference unchanged. Otherwise, it securely aggregates the accepted encrypted coordinates, decrypts only the aggregate coordinates at the TKA, forms the signed average global gradient using the fixed-point scale S, updates the model, then performs a momentum-based reference refresh and normalization before storing the new reference at the FIP Verifier and broadcasting the next model.

Figure. Editable academic-style geometric diagram for Algorithm 5. The server first checks whether the accepted encrypted set 𝒜(t) is empty. If so, it keeps the current model and trusted reference unchanged, broadcasts θ(t+1) = θ(t), and returns the unchanged pair. Otherwise it sets m = |𝒜(t)| and, for each coordinate k, initializes an encrypted aggregate accumulator [[Ĝk(t)]] ← Encpk(0). It then homomorphically combines the accepted encrypted encoded coordinates by [[Ĝk(t)]] ← [[Ĝk(t)]] · [[ĝi,k(t)]] mod n2. Only the encrypted aggregate coordinates are sent to the TKA, where Ĝk(t) ← Decsk([[Ĝk(t)]]) and the average global gradient is recovered by gglobal,k(t) ← DecodeS(Ĝk(t))/m, where DecodeS(·) maps Paillier plaintexts back to signed fixed-point values. The global model is then updated by θ(t+1) ← θ(t) − ηgglobal(t). Finally, the trusted reference is refreshed with momentum as Ṙ(t+1) ← βR(t) + (1−β)gglobal(t), and normalized as R(t+1) ← Ṙ(t+1) / max(‖Ṙ(t+1)‖2, ε). The refreshed reference is stored at the FIP Verifier, and the updated model is broadcast for the next round.
